GLFM provides a minimal, event-driven C API for building OpenGL ES apps on iOS, tvOS, Android, and WebGL. Inspired by GLFW, it handles display setup, touch and keyboard input, hardware sensors, and application lifecycle — so you can focus on your app's rendering logic.

## Key features

- **OpenGL ES 2, 3, 3.1, 3.2** and **Metal** display setup
- **Retina / high-DPI** support across all platforms
- **Touch, mouse, keyboard**, and mouse wheel input events
- **Hardware sensors**: accelerometer, magnetometer, gyroscope, rotation matrix
- **Application lifecycle**: focus, memory warnings, surface created/destroyed
- **Virtual keyboard** and **clipboard** on iOS and Android
- **Haptic feedback** on iOS (iPhone 7+) and Android
- **Safe area insets** ("chrome insets") for notched devices

<Note>
GLFM is intentionally limited in scope. It does not provide image loading, text rendering, audio, or UI toolkits. Use it alongside other cross-platform libraries for those needs.
</Note>

## How it works

GLFM is event-driven: you implement `glfmMain()` as your entry point, configure the display, register callback functions, and GLFM manages the platform's run loop. This model is required on iOS (where `UIApplicationMain` never returns) and WebGL (which is inherently event-driven).

```c glfmMain entry point
#include "glfm.h"

void glfmMain(GLFMDisplay *display) {
    glfmSetDisplayConfig(display,
                         GLFMRenderingAPIOpenGLES2,
                         GLFMColorFormatRGBA8888,
                         GLFMDepthFormatNone,
                         GLFMStencilFormatNone,
                         GLFMMultisampleNone);
    glfmSetRenderFunc(display, onDraw);
}
```
